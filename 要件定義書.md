```markdown
# 要件定義書

## 1. 背景・目的
- **背景**: 企業向けAI/プログラミング研修の市場拡大に伴い、学習管理を効率化・可視化するLMSが必要
- **目的**: 受講者の学習効率向上と管理者の運用負荷軽減を実現

## 2. 対象ユーザー
- **システム管理者**: 全体設定、権限管理
- **コース管理者**: 自身担当コースの作成・運用
- **講師**: カリキュラム作成・課題レビュー
- **受講者**: 自身のコース閲覧・受講

## 3. 機能要件
### 3.1 認証・権限管理
- メールアドレス＋パスワード認証（JWT）
- ロール: システム管理者／コース管理者／講師／受講者
- パスワード強度チェック、アカウントロック
- ダークモード切替

### 3.2 コース管理
- コースCRUD（タイトル、説明、タグ、公開ステータス）
- カテゴリ・タグ設定
- コースバージョン管理（最新版のみ保持）

### 3.3 カリキュラム管理
- 階層構造: モジュール → レッスン
- コンテンツタイプ: テキスト/Markdown、PDF埋め込み、スライド
- 並び替え、下書き保存

### 3.4 受講者管理
- 一括(CSV)/個別登録
- ステータス: 支払い待ち/受講開始待ち/受講中/解約/卒業
- コース付与・削除

### 3.5 学習進捗管理
- 既読マーク、読了タイムスタンプ
- 課題提出／レビュー承認管理
- レポート: 読了率、提出率、期間

### 3.6 通知／リマインド
- イベント発火: 登録/開始/読了/提出/レビュー完了 など
- チャネル: メール／Web通知／Slack(Webhook)
- テンプレート管理UI（件名・本文編集）

### 3.7 連携
- Salesforce API、Slack API、Googleサービス(Gmail/Drive/Sheets)
- Webhooks (外部システム連携)

## 4. 非機能要件
- **セキュリティ**: SSL/TLS、データ暗号化、MFA検討
- **性能**: 初期100名規模、将来1000+想定
- **可用性**: 99.9%目標
- **運用性**: ログ取得、監視アラート
- **対応環境**: レスポンシブWeb、ダークモード
```
